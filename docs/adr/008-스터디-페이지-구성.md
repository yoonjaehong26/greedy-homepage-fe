# ADR 008. 스터디 페이지 구성

- 상태: 확정 (2026-07, 구현 #16을 소급 기록)
- 결정자: 4기 홈페이지 TF

## 맥락

스터디는 그리디 활동의 중심이에요. 지망 부원이 "들어가면 뭘 어떻게 배우는지"를 그려볼 수 있어야 하고, 트랙(프론트엔드/백엔드)마다 커리큘럼이 달라서 자기 트랙 기준으로 읽을 수 있어야 해요.

## 결정

한 화면(`/study`)에 위에서 아래로 둬요.

1. **페이지 헤더** - "스터디" + 매주 미션을 만들고, 리뷰로 다듬으며 성장해요.
2. **트랙 탭** - 프론트엔드 / 백엔드. 탭을 바꾸면 트랙 소개와 커리큘럼이 함께 바뀌어요.
3. **트랙 소개** - 무엇을 만드는 트랙인지 + 다루는 기술 배지.
4. **한 주의 리듬** - 미션 받기 → 구현 → 티키타카 리뷰 → 머지. 두 트랙이 같아서 탭과 무관하게 고정이에요.
5. **주차별 커리큘럼** - `TimelineItem`으로 페이즈를 세로로 늘어놓아요. 페이즈마다 주차 범위·미션·설명. 하단에 "커리큘럼은 기수마다 조금씩 달라져요. 4기 기준이에요." 주석.

요일과 마감 시각은 넣지 않아요. 들어온 뒤 공지로 알면 되는 운영 정보라 밖에 알릴 필요가 없고, 대신 주차 범위로 흐름만 보여줘요.

## 이유

- **트랙을 탭으로.** 두 커리큘럼을 나란히 두면 자기 트랙을 찾아 읽어야 해요. 탭이면 처음부터 한 트랙 기준으로 읽혀요.
- **리듬은 고정.** 한 주가 굴러가는 방식은 두 트랙이 같아서, 탭 안에 넣으면 같은 내용이 두 번 있는 것처럼 보여요.
- **날짜 대신 주차.** 요일·마감은 기수 운영에 따라 바뀌는 값이라 사이트에 박으면 금방 낡아요. 주차 범위는 커리큘럼의 구조라 오래가요.

## 코드 배치

- 스터디 페이지에서만 쓰는 구획이라 `app/study/_sections`에 라우트와 같이 둬요 ([ADR 005](005-폴더-구조-축소형-FSD.md)).
- 문구와 커리큘럼은 서버가 아니라 기수마다 손으로 고치는 콘텐츠라 `app/study/_sections/content.ts`에 둬요 ([ADR 003](003-콘텐츠-코드-분리.md)).
- 탭 상태만 client(`StudyTracks`)이고 나머지는 서버에서 렌더해요.
